show mpls l2 vc 5724412 detail --> Detalles del VC (Estado y último cambio y algún tunnel)
show mpls l2transport  vc 502127501  detail | in change <--- para ver ultimo cambio en el vc 
show xconnect all | in 5724412  --> Estado del pseudowire y la interface física 
show interfaces Gi0/0/3 description --> Estado de la interface física 
show ethernet service instance id 1017 interface Te0/0/4 stats  -> Para ver si hay paquetes de salida o de entrada en la itx o en la aggre 
show interfaces Gi0/0/3 transceiver detail --> Potencias en la interface
show interfaces Gi0/0/3 human-readable | in error|drops|rate --> Aumeto de errores y el tráfico por la interface

show ethernet service instance id 2597 interface Gi0/0/7 stats <--- para ver entrada y salida de paquetes  
show ethernet service instance id 2441 interface Gi0/0/3 stats  --> Aumento de paquetes en ambos sentidos 
show mpls traffic-eng tunnels tunnel 5657 --> información del tunnel. Si está arriba y los últimos cambios.

Huawei Grande — Core / EDGE
Comandos de BVI, ARP, ping VPN, VSI y mpls
Display interface description | include COCOVER10107377I <-- para ver la información de ID 
display current-configuration interface Vbdif2641 (Esto es igual a una BVI -Vbdif-) <-  Esto es para ver el pool de IPs del cliente 
display arp vpn-instance INTERNET  slot 0 dynamic | include Vbdif2641   <- para ver aprendizaje ARP 
ping -vpn-instance INTERNET -brief -c 10 -s 1000 -m 20 -a  181.78.78.36  8.8.8.8 <- Para hacer la prueba de ping 

display current-configuration | be COCOORA10010725I --> Me da la conf de las Vbdif (Es como la BVI en Cisco)
display interface description | in COCOORA10010725I --> Me da el estado de la interface Vbdif
display interface Vbdif 565 --> Adicional a la conf me dice el MTU, tiempo en línea y los rate (BVI en cisco)
display arp vpn-instance INTERNET  slot 0 dynamic | include Vbdif565 --> Nos dice la tabla ARP que aprendemos para los pool de IPs que se tienen configuradas.
ping -vpn-instance INTERNET 138.186.21.182 ip-forwarding --> ping a la IP del cliente
ping -vpn-instance INTERNET 8.8.8.8 --> INTERNET o cualquier IP
telnet vpn-instance INTERNET  (IP)  80  | abrir puerto 80 
display ip routing-table vpn-instance INTERNET 8.8.8.8 | ver ruta a internet 
display current-configuration | section include 565 --> Este me da la VSI que es como el pw o pseudowire en los cisco (me da el peer del que venimos y el VC)
display vsi name VLAN565 verbose --> Nos dice el estado del VSI y si hubo cambios recientes asi como la creación del mismo 
display vsi name VLAN565 peer-info --> Nos dice la info del peer y el estado más organizado 
display mpls te tunnel | in 172.17.28.152 o 5756501 --> Con este vemos si hay un tunnel asociado a este VC 
ping vpls vsi VLAN565 peer 172.17.28.152 --> Se hace ping al peer (Similar al ping mpls pseudowire)
display ospf peer brief --> Ver vecinos ospf
